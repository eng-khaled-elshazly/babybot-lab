## ERR-007 | ORA-00904 Mixed-Case Requires Quotes

```
Error:    ORA-00904: "DATE_INSERT_DB": invalid identifier
          (when SQL Server column is actually "date_insert_db" in mixed case)
Cause:    Oracle/DG4ODBC uppercases unquoted identifiers when querying through DB Link
          SQL Server columns with mixed case (e.g., "date_insert_db", "ModifiedDate")
          are accessed as uppercase "DATE_INSERT_DB" and SQL Server rejects them
Fix:      Quote ALL column names in Oracle View SQL to preserve exact case
Example:
  -- Wrong (gets uppercased to DATE_INSERT_DB)
  SELECT date_insert_db FROM dbo.GL_Accounts@MSSQL_LINK_2019

  -- Correct (preserves exact case)
  SELECT "date_insert_db" FROM "dbo"."GL_Accounts"@MSSQL_LINK_2019
Prevention:
  - Always quote column names in View SQL when SQL Server uses mixed case
  - Quote schema and table names too: "dbo"."GL_Accounts"
  - Test pattern: sqlplus> SELECT "column_name" FROM ...
  - Update View generator to add quotes automatically for all DB Link views
Date:     2026-03-11
Status:   RESOLVED
```
